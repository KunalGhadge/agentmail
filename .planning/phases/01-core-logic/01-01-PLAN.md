# Plan 01-01: Core Logic & Cryptography

This plan covers the foundational identity and storage systems for AgentMail.

## Tasks
- [ ] Create `requirements.txt` with `mcp` and `cryptography`.
- [ ] Implement `identity.py`: RSA-PSS key generation and signing.
- [ ] Implement `storage.py`: Repository Pattern for local JSON storage.
- [ ] Manual Verification: Run a test script to sign a payload and save/retrieve it.

## Implementation Details

### Identity System (`identity.py`)
- Folder: `.agent_keys/`
- Files: `pub.pem`, `priv.pem`, `.agent_id`
- Logic: Check for `.agent_id`. If missing, generate everything.
- Signature: RSA-PSS + SHA-256 + PKCS1v11 padding (or standard MGF1).

### Storage System (`storage.py`)
- File: `inbox.json`
- Pattern: Repository Pattern with atomic writes.
- Format: `{"id": UUID, "to": UUID, "from": UUID, "payload": JSON, "signature": B64, "timestamp": UTC}`.
